# Security Policy

Perspective Room bundles can include fundraising materials, claims, founder contact details, and links to sensitive documents.

## Reporting A Security Issue

Please report security issues privately to the repository maintainer before opening a public issue.

Include:

- affected version or commit
- the files or fields involved
- reproduction steps
- expected impact

## Sensitive Data Guidance

Do not include confidential company materials, investor lists, customer names, financial models, credentials, tokens, or unreleased company data in public issues, pull requests, examples, or eval fixtures.

Use invented company data in examples.

For hosted rooms, remember that a static bundle is portable but not permissioned. Anyone with access to the hosted bundle can access included files.
