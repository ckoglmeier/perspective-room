# Investor Relations

## Overview

Use this skill as the operating layer for a fundraising story. Maintain consistency across artifacts, preserve source-of-truth boundaries, and keep investor-facing claims proportional to the evidence.

## Canonical Source Map

Start by asking or inferring which artifacts exist:

- Current investor deck.
- Fundraising memo or narrative doc.
- Financial model.
- Investor room / Perspective Room export.
- FAQ and diligence answers.
- Customer proof and contracts.
- Benchmark/eval/proof docs.
- Source notes and changelog.

If artifacts disagree, do not silently smooth over conflicts. Build a claim ledger and identify which source should win.

## Source Precedence

Use project-specific precedence if the user provides it. Otherwise default to:

1. Financial model wins for formulas, scenario outputs, cash, runway, pricing mechanics, and forecast math.
2. Latest investor deck wins for current external narrative and round ask.
3. Proof docs win for benchmark methodology, customer proof, experiment scope, and caveats.
4. Memo/FAQ wins for diligence posture, risk framing, and investor objections.
5. Room export wins only for what has already been packaged, not for underlying truth.

## Coordination Workflow

1. Identify the target artifact and audience.
2. Read relevant canonical sources before drafting or editing.
3. Build a claim ledger with `claim`, `current source`, `conflicting source`, and `action`.
4. Resolve conflicts by source precedence or mark them as open decisions.
5. Draft or edit in a concrete, evidence-backed investor voice.
6. Propagate material changes to companion artifacts or leave a visible reconciliation note.
7. Verify dates, numbers, round terms, customer proof, market claims, and technical claims before strengthening language.

## Common Tasks

- Investor deck consistency audit.
- Fundraising memo rewrite.
- Model narrative explanation.
- Investor update drafting.
- Data room / Perspective Room refresh.
- Diligence FAQ creation.
- Claim-strength audit.
- Round narrative cleanup.

## Output Patterns

For a consistency audit:

- Executive read.
- Source-of-truth table.
- Conflicts and recommended fixes.
- Required edits by artifact.
- Founder decisions needed.

For drafting:

- Revised copy.
- Claim ledger for strong claims.
- Companion updates needed in other artifacts.

For room maintenance:

- Changed room fields.
- Stale/missing materials.
- Founder decisions needed.
