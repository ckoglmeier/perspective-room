# Fundraising Claim Auditor

## Overview

Use this skill to protect credibility in fundraising materials. The goal is not to make claims smaller; it is to make them exactly as strong as the evidence allows.

## Workflow

1. **Collect evidence**
   - Identify the artifact containing the claim and the source material supporting it.
   - Capture date, sample, scope, baseline, customer/source status, and caveats.

2. **Classify the claim**
   - Use `proven`, `shown`, `suggested`, `not_shown`, `contradicted`, or `unsupported`.
   - Distinguish direct evidence from founder interpretation.

3. **Check fundraising overreach**
   - One customer -> market proof.
   - Pilot -> repeatable GTM.
   - Static eval -> production robustness.
   - Benchmark win -> product adoption.
   - LOI/verbal interest -> contracted revenue.
   - Pipeline -> revenue.
   - Generated room summary -> verified source truth.

4. **Rewrite**
   - Preserve the commercial insight.
   - Add scope boundaries.
   - Use words like "shows", "suggests", "points toward", and "does not yet show" precisely.
   - Remove vague superlatives unless sourced.

5. **Define the next proof**
   - State the smallest evidence that would upgrade the claim.
   - Include what metric, baseline, customer, or repeatability proof is missing.

## Output Shape

Return:

- Bottom line.
- Claim audit table.
- Investor-safe rewrite.
- Residual caveats.
- Next proof needed.

For deck edits, provide replacement copy short enough to fit the slide.
