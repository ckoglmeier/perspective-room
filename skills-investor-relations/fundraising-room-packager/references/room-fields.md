# Suggested Room Fields

Use these fields when building a structured fundraising room. Adapt to the target schema when one exists.

- `company`: name, website, one-liner, category.
- `round`: stage, amount, instrument, valuation/cap if disclosed, status.
- `narrative`: problem, insight, solution, wedge, why now.
- `traction`: revenue, customers, usage, pipeline, pilots, retention, growth.
- `proof`: customer proof, benchmark proof, technical proof, market proof.
- `team`: founders, relevant background, gaps.
- `business_model`: pricing, sales motion, expansion path, gross margin or cost structure when available.
- `materials`: file manifest with title, type, path/link, access mode, date, owner.
- `claims`: claim, source, status, confidence, date, caveat.
- `founder_decisions_needed`: missing materials, known risks, and follow-ups the founder should resolve before sharing.
- `agent_notes`: parsing hints, sensitive boundaries, source-of-truth notes.
